## Historia de Usuario

**Como** usuario,

**Quiero** buscar registros en una entidad relacionada escribiendo,

**Para** encontrar rápidamente el registro correcto cuando hay muchos.

## Descripción

El endpoint GET /api/entities/{id}/lookup permite buscar registros de una entidad por un término de búsqueda. Se usa para alimentar el dropdown/autocomplete de campos RELATION. El usuario escribe en el campo y los resultados se filtran según el display_field (o campos de texto de la entidad). El dropdown muestra los resultados filtrados mientras el usuario escribe.

## Criterios de Aceptación

```gherkin
Scenario: Endpoint lookup retorna registros filtrados
  Given que existe la entidad "Clientes" con display_field "nombre"
  And existen clientes "Juan Pérez", "María García", "Juan López"
  When hago GET /api/entities/{clientes_id}/lookup?search=Juan
  Then la respuesta incluye los registros que coinciden con "Juan"
  And cada registro tiene id y el valor del display_field
  And la respuesta incluye "Juan Pérez" y "Juan López"

Scenario: Lookup sin search retorna lista paginada
  Given que hago GET /api/entities/{id}/lookup sin parámetro search
  When la entidad tiene registros
  Then retorna los primeros N registros (ej: 20)
  And la respuesta tiene formato { items: [...], total: N }

Scenario: Dropdown filtra resultados al escribir
  Given que el usuario está en el campo RELATION "cliente"
  When escribe "Mar" en el campo
  Then se hace una petición a /lookup?search=Mar
  And el dropdown muestra solo los registros que coinciden
  And al escribir más caracteres, se actualiza la búsqueda (debounce)

Scenario: Lookup respeta permisos del usuario
  Given que soy un usuario con acceso a la entidad
  When hago GET /api/entities/{id}/lookup
  Then solo recibo registros que tengo permiso de leer
  And si no tengo acceso a la entidad, recibo 403

Scenario: Lookup con entidad inexistente retorna 404
  Given que el entity_id no existe
  When hago GET /api/entities/{invalid_id}/lookup
  Then recibo un error 404
```

## Notas Adicionales

- Endpoint: `GET /api/entities/{entity_id}/lookup?search={term}&limit=20`
- Parámetros opcionales: search (filtra por display_field y/o campos texto), limit (default 20)
- Respuesta: `{ "items": [{ "id": "uuid", "display_value": "valor" }], "total": N }`
- Implementar debounce en el frontend (300-500ms) para no saturar con requests al escribir
- La búsqueda puede usar ILIKE o similar sobre el display_field
