# US-059: Eliminar relación entre entidades

## Información General

| Campo | Valor |
|-------|-------|
| **ID** | US-059 |
| **Título** | Eliminar relación entre entidades |
| **Épica** | EP-09: Relaciones entre Entidades |
| **Prioridad** | P1 (Alta) |
| **Estimación** | 30 min |
| **Dependencias** | US-054, US-055, US-058 |

## Historia de Usuario

**Como** administrador,

**Quiero** eliminar una relación,

**Para** limpiar referencias que ya no se utilizan.

## Descripción

El administrador puede eliminar una relación entre entidades. La eliminación de un campo tipo RELATION debe eliminar también el registro en entity_relationships y ejecutar un ALTER TABLE para eliminar (DROP) la columna correspondiente de la tabla dinámica. Si la relación se gestiona desde la sección de relaciones, la eliminación debe eliminar el campo asociado y la columna.

## Criterios de Aceptación

```gherkin
Scenario: Eliminar campo RELATION elimina relación y columna
  Given que existe un campo "cliente" tipo RELATION en "Órdenes"
  When el Admin elimina el campo "cliente"
  Then se elimina el registro en entity_relationships
  And se ejecuta ALTER TABLE DROP COLUMN cliente
  And el campo desaparece de la entidad

Scenario: Eliminar relación desde sección Relaciones
  Given que estoy en el detalle de la entidad viendo la sección Relaciones
  When hago clic en "Eliminar" en una relación
  Then aparece modal de confirmación
  And al confirmar, se elimina el campo RELATION asociado
  And se elimina la entrada en entity_relationships
  And se hace DROP de la columna

Scenario: Confirmación antes de eliminar
  Given que voy a eliminar una relación
  When hago clic en Eliminar
  Then se muestra un modal de confirmación
  And debo confirmar para proceder
  And al cancelar, no se realiza ningún cambio

Scenario: Datos existentes en la columna
  Given que existen registros con valores en el campo RELATION
  When elimino la relación
  Then la columna se elimina (los datos se pierden)
  And se muestra advertencia en el modal sobre pérdida de datos

Scenario: Solo Admin puede eliminar relaciones
  Given que soy un usuario con rol User
  When intento eliminar una relación
  Then recibo un error 403 Forbidden
```

## Notas Adicionales

- La eliminación es en cascada: eliminar campo RELATION → eliminar entity_relationship → DROP COLUMN
- Modal de confirmación debe advertir sobre pérdida de datos en la columna
- ALTER TABLE: `DROP COLUMN {field_name}`
- Validar que no existan constraints que impidan el DROP (ej: índices, FKs)
