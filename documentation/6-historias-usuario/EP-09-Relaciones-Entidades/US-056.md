## Historia de Usuario

**Como** usuario,

**Quiero** ver un selector dropdown para los campos RELATION en el formulario CRUD,

**Para** poder seleccionar registros relacionados fácilmente.

## Descripción

Cuando un formulario CRUD dinámico incluye un campo tipo RELATION, el DynamicForm debe renderizarlo como un autocomplete o dropdown en lugar de un input de texto. El dropdown muestra los valores del display_field de la entidad destino, permitiendo al usuario seleccionar el registro relacionado. El valor guardado es el UUID del registro seleccionado.

## Criterios de Aceptación

```gherkin
Scenario: DynamicForm renderiza RELATION como dropdown/autocomplete
  Given que la entidad "Órdenes" tiene un campo "cliente" tipo RELATION
  And el display_field es "nombre"
  When abro el formulario de crear/editar orden
  Then el campo "cliente" se renderiza como dropdown o autocomplete
  And no se muestra como input de texto simple

Scenario: Dropdown muestra valores del display_field
  Given que el campo RELATION apunta a entidad "Clientes" con display_field "nombre"
  And existen clientes "Juan Pérez", "María García", "Carlos López"
  When abro el dropdown del campo cliente
  Then veo las opciones "Juan Pérez", "María García", "Carlos López"
  And cada opción corresponde al UUID del registro

Scenario: Usuario puede seleccionar un registro
  Given que el dropdown muestra los clientes disponibles
  When selecciono "María García"
  Then el valor del campo se establece al UUID de ese registro
  And al guardar el formulario, se persiste el UUID correctamente

Scenario: Campo RELATION opcional muestra opción vacía
  Given que el campo RELATION no es requerido
  When abro el dropdown
  Then la primera opción es "-- Seleccionar --" o vacío
  And el usuario puede dejar el campo sin selección

Scenario: Al editar, se muestra el valor actual seleccionado
  Given que un registro tiene cliente_id = uuid-de-maria
  When abro el formulario de edición
  Then el dropdown muestra "María García" como valor seleccionado
  And el usuario puede cambiar la selección
```

## Notas Adicionales

- Usar componente autocomplete o select con búsqueda para mejor UX (especialmente con muchos registros)
- El backend debe exponer endpoint de lookup (US-057) para cargar opciones
- Considerar paginación si la entidad destino tiene muchos registros
- El formulario envía el UUID en el payload, no el valor del display_field
