# US-058: Listar relaciones de una entidad

## Información General

| Campo | Valor |
|-------|-------|
| **ID** | US-058 |
| **Título** | Listar relaciones de una entidad |
| **Épica** | EP-09: Relaciones entre Entidades |
| **Prioridad** | P1 (Alta) |
| **Estimación** | 30 min |
| **Dependencias** | US-054, US-055 |

## Historia de Usuario

**Como** administrador,

**Quiero** ver todas las relaciones definidas para una entidad,

**Para** poder gestionarlas.

## Descripción

En la página de detalle de una entidad (Entity detail), el administrador puede ver una sección "Relaciones" que lista todas las relaciones en las que la entidad participa, ya sea como origen (source) o como destino (target). Cada relación muestra el nombre del campo (si aplica), la entidad relacionada y el tipo de relación. Esto facilita la gestión y comprensión del modelo de datos.

## Criterios de Aceptación

```gherkin
Scenario: Entity detail muestra sección Relaciones
  Given que soy Admin y accedo al detalle de la entidad "Órdenes"
  When la entidad tiene relaciones definidas
  Then veo una sección "Relaciones" en la página
  And la sección lista las relaciones existentes

Scenario: Se listan relaciones como source
  Given que "Órdenes" tiene un campo "cliente" RELATION hacia "Clientes"
  When veo la sección Relaciones
  Then aparece la relación: Órdenes.cliente -> Clientes (MANY_TO_ONE)
  And se muestra el nombre del campo y la entidad destino

Scenario: Se listan relaciones como target (inversas)
  Given que "Clientes" es target de la relación Órdenes.cliente
  When veo el detalle de "Clientes"
  Then aparece que "Órdenes" referencia a esta entidad
  And se muestra el campo "cliente" de la entidad Órdenes

Scenario: Entidad sin relaciones muestra mensaje
  Given que la entidad "Productos" no tiene relaciones
  When veo la sección Relaciones
  Then se muestra "No hay relaciones definidas"
  And opcionalmente un enlace para agregar relación

Scenario: Solo Admin ve la sección Relaciones
  Given que soy un usuario con rol User
  When accedo al detalle de una entidad
  Then no veo la sección de Relaciones (o está oculta)
```

## Notas Adicionales

- La sección puede ser una pestaña "Relaciones" o un bloque colapsable
- Mostrar: nombre campo, entidad origen, entidad destino, tipo relación
- Considerar tabla o lista de cards según diseño
- Endpoint: puede incluirse en GET /api/entities/{id} o GET /api/entities/{id}/relationships
