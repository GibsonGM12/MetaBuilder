# US-054: Definir relación entre dos entidades

| **Estimación** | 90 min |
| **Dependencias** | US-016, US-017 |

## Historia de Usuario

**Como** administrador,

**Quiero** definir una relación entre dos entidades,

**Para** que los datos puedan referenciar registros de otras entidades.

## Descripción

El administrador puede crear relaciones entre entidades del sistema. Una relación especifica la entidad origen (source), la entidad destino (target) y el tipo de relación. En esta versión se soporta MANY_TO_ONE (muchos registros de la entidad origen pueden referenciar un mismo registro de la entidad destino). La relación se persiste en la tabla `entity_relationships`.

## Criterios de Aceptación

```gherkin
Scenario: Admin define relación MANY_TO_ONE exitosamente
  Given que soy un usuario con rol Admin
  And existen las entidades "Órdenes" y "Clientes"
  When creo una relación con source="Órdenes", target="Clientes", type=MANY_TO_ONE
  Then se crea un registro en entity_relationships
  And la relación tiene id, source_entity_id, target_entity_id, relationship_type

Scenario: Admin selecciona entidad origen
  Given que estoy creando una relación
  When selecciono la entidad origen
  Then veo un dropdown con todas las entidades del sistema
  And no puedo seleccionar la misma entidad como origen y destino

Scenario: Admin selecciona entidad destino
  Given que seleccioné la entidad origen "Órdenes"
  When selecciono la entidad destino
  Then veo un dropdown con las entidades disponibles (excluyendo la origen)
  And selecciono "Clientes"

Scenario: Tipo de relación es MANY_TO_ONE
  Given que estoy definiendo una relación
  When selecciono el tipo de relación
  Then MANY_TO_ONE es la única opción disponible (por ahora)
  And la relación se guarda correctamente

Scenario: Solo Admin puede definir relaciones
  Given que soy un usuario con rol User
  When intento crear una relación entre entidades
  Then recibo un error 403 Forbidden
```

## Notas Adicionales

- La relación se almacena en la tabla `entity_relationships` con columnas: id, source_entity_id, target_entity_id, relationship_type, created_at
- relationship_type por defecto: MANY_TO_ONE
- Validar que source_entity_id ≠ target_entity_id
- Una relación se vincula posteriormente a un campo tipo RELATION (US-055)
