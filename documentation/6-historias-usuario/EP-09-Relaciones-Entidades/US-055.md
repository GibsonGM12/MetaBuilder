## Historia de Usuario

**Como** administrador,

**Quiero** agregar un campo tipo RELATION a una entidad,

**Para** que pueda almacenar una referencia a registros de otra entidad.

## Descripción

El administrador puede crear campos de tipo RELATION en una entidad. Al seleccionar este tipo en el Field Manager, aparecen controles adicionales: selector de entidad destino (target entity) y selector de campo de visualización (display field). Al guardar, se ejecuta un ALTER TABLE que agrega una columna UUID a la tabla dinámica para almacenar el ID del registro relacionado.

## Criterios de Aceptación

```gherkin
Scenario: RELATION aparece en dropdown de tipos de campo
  Given que estoy agregando un campo a una entidad
  When abro el selector de tipo de campo
  Then "RELATION" aparece en la lista junto a TEXT, NUMBER, DATE, etc.

Scenario: Al seleccionar RELATION aparecen controles adicionales
  Given que seleccioné tipo RELATION
  When se muestra el formulario de configuración
  Then aparece un selector "Entidad destino" con las entidades disponibles
  And aparece un selector "Campo de visualización" (display field)
  And el display field muestra los campos de la entidad destino seleccionada

Scenario: Display field determina qué se muestra en el dropdown
  Given que la entidad destino "Clientes" tiene campos nombre, email, telefono
  When selecciono "nombre" como display field
  Then en el CRUD se mostrarán los valores de nombre en el dropdown

Scenario: ALTER TABLE agrega columna UUID
  Given que guardo un campo RELATION en la entidad "Órdenes"
  When se ejecuta el ALTER TABLE
  Then se agrega una columna UUID (o equivalente) a la tabla entity_xxx
  And la columna permite NULL para registros sin relación
  And se crea/actualiza el registro en entity_relationships

Scenario: Campo RELATION requiere entidad destino y display field
  Given que estoy creando un campo RELATION
  When no especifico target_entity_id o display_field
  Then recibo un error 422 con mensaje descriptivo
```

## Notas Adicionales

- El campo RELATION almacena el UUID del registro de la entidad destino
- ALTER TABLE: `ADD COLUMN {field_name} UUID REFERENCES entity_{target_uuid}(id)` o equivalente
- display_field: nombre del campo de la entidad destino que se usa para mostrar en dropdowns (ej: "nombre")
- El campo se vincula a una relación en entity_relationships (source=entidad actual, target=entidad seleccionada)
