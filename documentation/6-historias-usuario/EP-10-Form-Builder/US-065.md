# US-065: Diseñar layout del formulario

## Historia de Usuario

**Como** administrador,

**Quiero** arrastrar y reordenar las secciones del formulario visualmente,

**Para** definir el orden en que aparecerán los campos cuando el usuario llene el formulario.

## Descripción

El administrador puede diseñar el layout del formulario arrastrando las secciones para cambiar su orden. Las secciones se muestran en un canvas con drag & drop (similar al Dashboard Designer). El orden se persiste en sections_config como array ordenado. Cada sección muestra su tipo y título para identificación visual.

## Criterios de Aceptación

```gherkin
Scenario: Admin reordena secciones con drag and drop
  Given que el formulario tiene secciones: Cabecera, Cliente, Detalle, Totales
  When arrastro "Totales" antes de "Detalle"
  Then el orden visual se actualiza
  And al guardar, sections_config refleja el nuevo orden
  And el usuario ve las secciones en ese orden al llenar

Scenario: Secciones son arrastrables en el designer
  Given que estoy en el Form Designer
  When veo la lista de secciones
  Then cada sección tiene handle de arrastre
  And puedo soltar en cualquier posición
  And hay feedback visual durante el arrastre

Scenario: Orden por defecto es orden de creación
  Given que agrego secciones una por una
  When no reordeno manualmente
  Then el orden es el de creación (primera agregada = primera en lista)
  And sections_config mantiene ese orden

Scenario: Eliminar sección la remueve del layout
  Given que tengo una sección en el formulario
  When elimino la sección desde el designer
  Then la sección desaparece del canvas
  And se actualiza sections_config
  And el usuario ya no ve esa sección al llenar
```

## Notas Adicionales

- Usar librería de drag & drop (ej: @dnd-kit, react-beautiful-dnd) o similar al Dashboard Designer
- sections_config: array de objetos, cada uno con type, config, order/index
- El orden del array determina el orden de renderizado
