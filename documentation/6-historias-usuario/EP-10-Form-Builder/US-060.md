# US-060: Crear formulario personalizado

|-------|-------|

| **Título** | Crear formulario personalizado |
| **Épica** | EP-10: Form Builder |
| **Prioridad** | P0 (Crítica) |
| **Estimación** | 60 min |
| **Dependencias** | US-016, US-017 |

## Historia de Usuario

**Como** administrador,

**Quiero** crear un nuevo formulario personalizado con nombre, descripción y entidad principal,

**Para** tener un contenedor donde diseñar secciones de captura de datos que abarquen múltiples entidades.

## Descripción

El administrador puede crear formularios personalizados vacíos que servirán como contenedores para secciones. Cada formulario tiene un nombre único, una descripción opcional y una entidad principal (primary entity). Los metadatos se almacenan en la tabla `forms`. Solo los administradores pueden crear formularios.

## Criterios de Aceptación

```gherkin
Scenario: Admin crea formulario exitosamente
  Given que soy un usuario con rol Admin
  When envío un POST a /api/forms con nombre "Captura de Venta", descripción "Formulario para ventas" y primary_entity_id
  Then se crea el formulario con un UUID
  And la respuesta incluye id, name, description, primary_entity_id, created_at

Scenario: User no puede crear formulario
  Given que soy un usuario con rol User
  When intento crear un formulario
  Then recibo un error 403 Forbidden

Scenario: Nombre y entidad principal son requeridos
  Given que soy Admin
  When intento crear un formulario sin nombre o sin primary_entity_id
  Then recibo un error 422 con mensaje descriptivo

Scenario: Entidad principal debe existir
  Given que soy Admin
  When intento crear un formulario con primary_entity_id inexistente
  Then recibo un error 404 o 422 con mensaje descriptivo
```

## Notas Adicionales

- El formulario se crea vacío (sin secciones)
- La entidad principal define el contexto del formulario (ej: Ventas, Órdenes)
- Se almacena el `created_by` con el ID del usuario que lo crea
- La tabla `forms` contiene: id, name, description, primary_entity_id, sections_config (JSON), created_at, updated_at
