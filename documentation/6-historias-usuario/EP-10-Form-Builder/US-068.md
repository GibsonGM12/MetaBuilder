# US-068: Editar y eliminar formulario

## Información General

| Campo | Valor |
|-------|-------|
| **ID** | US-068 |
| **Título** | Editar y eliminar formulario |
| **Épica** | EP-10: Form Builder |
| **Prioridad** | P1 (Alta) |
| **Estimación** | 30 min |
| **Dependencias** | US-060, US-065 |

## Historia de Usuario

**Como** administrador,

**Quiero** editar la estructura de un formulario o eliminarlo,

**Para** mantener los formularios actualizados o remover los que ya no se usan.

## Descripción

El administrador puede editar un formulario existente: cambiar nombre, descripción, entidad principal, agregar/eliminar/reordenar secciones. También puede eliminar un formulario. La eliminación puede ser soft delete (marcar como inactivo) o hard delete según decisión de implementación. Solo los administradores tienen acceso a estas acciones.

## Criterios de Aceptación

```gherkin
Scenario: Admin edita formulario exitosamente
  Given que soy Admin y tengo un formulario "Captura de Venta"
  When edito el nombre a "Captura de Venta v2" y agrego una sección
  And hago clic en "Guardar"
  Then los cambios se persisten
  And GET /api/forms/:id devuelve los datos actualizados

Scenario: Admin elimina formulario
  Given que soy Admin y tengo un formulario
  When hago clic en "Eliminar" y confirmo en el modal
  Then el formulario se elimina (o marca inactivo)
  And ya no aparece en la lista de formularios
  And los registros creados previamente con el formulario permanecen (no cascade delete)

Scenario: User no puede editar ni eliminar
  Given que soy User
  When accedo a un formulario
  Then no veo botones "Editar" ni "Eliminar"
  And si intento PUT/DELETE por API recibo 403 Forbidden

Scenario: Editar abre Form Designer con datos cargados
  Given que hago clic en "Editar" en un formulario
  When se abre el Form Designer
  Then veo el nombre, descripción, secciones actuales
  And puedo modificar y guardar
  And el orden de secciones se mantiene o puedo reordenar
```

## Notas Adicionales

- PUT /api/forms/:id para actualizar
- DELETE /api/forms/:id para eliminar
- Considerar: formularios con submissions recientes podrían tener restricción de edición (solo agregar secciones, no eliminar) - opcional para v1
- Los registros creados vía formulario no se eliminan al borrar el formulario
