## Historia de Usuario

**Como** administrador,

**Quiero** agregar una sección LOOKUP a un formulario para seleccionar un registro de una entidad relacionada,

**Para** que los usuarios puedan elegir referencias (ej: cliente, producto) mediante un dropdown con búsqueda.

## Descripción

El administrador puede agregar una sección de tipo LOOKUP a un formulario. Esta sección permite seleccionar un registro de una entidad relacionada (relación MANY_TO_ONE). Se reutiliza el componente RelationLookup de EP-09 para mostrar un dropdown con búsqueda. El admin configura la entidad destino, el campo de visualización y la relación asociada.

## Criterios de Aceptación

```gherkin
Scenario: Admin agrega sección LOOKUP exitosamente
  Given que tengo un formulario con entidad principal "Ventas"
  And existe relación Ventas -> Clientes (MANY_TO_ONE)
  When agrego una sección LOOKUP con target_entity="Clientes", display_field="nombre"
  Then la sección se guarda en sections_config
  And la sección tiene type="LOOKUP", target_entity_id, display_field, relationship_id

Scenario: RelationLookup se reutiliza en el formulario
  Given que el formulario tiene una sección LOOKUP configurada
  When el usuario abre el formulario para llenar
  Then ve el componente RelationLookup (dropdown con búsqueda)
  And puede buscar y seleccionar un registro de la entidad destino
  And el valor seleccionado se asocia al campo de la entidad principal

Scenario: Lookup solo muestra entidades con relación válida
  Given que estoy agregando una sección LOOKUP
  When selecciono la entidad destino
  Then solo veo entidades que tienen relación MANY_TO_ONE con la entidad principal (o contexto)
  And selecciono la relación específica si hay múltiples

Scenario: Lookup opcional vs requerido
  Given que la sección LOOKUP tiene required=false
  When el usuario no selecciona ningún valor
  Then el formulario puede enviarse con el campo en NULL
  And si required=true, se valida antes de enviar
```

## Notas Adicionales

- Reutilizar RelationLookup de EP-09 (US-056, US-057)
- La sección LOOKUP mapea a un campo RELATION de la entidad principal
- Configuración: type, target_entity_id, display_field, relationship_id, required
