## Historia de Usuario

**Como** administrador,

**Quiero** agregar widgets a un dashboard existente,

**Para** configurar qué información se visualiza en el dashboard.

## Descripción

Desde el designer, el administrador puede agregar widgets de diferentes tipos a un dashboard. Cada widget se vincula a una entidad del sistema y tiene una configuración específica según su tipo. Los tipos de widget disponibles son: STAT_CARD, KPI_CARD, DATA_GRID, BAR_CHART, PIE_CHART, LINE_CHART, RECENT_LIST.

## Criterios de Aceptación

```gherkin
Scenario: Admin agrega widget a dashboard
  Given que tengo un dashboard creado
  When agrego un widget de tipo STAT_CARD vinculado a la entidad "Productos"
  Then el widget se crea con id, tipo, entity_id, posición y configuración
  And aparece en la lista de widgets del dashboard

Scenario: Widget requiere tipo y entidad
  Given que estoy agregando un widget
  When no especifico el widget_type o entity_id
  Then recibo un error 422

Scenario: Widget tiene posición por defecto
  Given que agrego un widget sin especificar posición
  Then se asigna posición automática {x: 0, y: 0, w: 4, h: 3}
```

## Notas Adicionales

- Cada widget debe especificar: widget_type, entity_id, title, position, config
- La posición usa formato `{x, y, w, h}` compatible con react-grid-layout
- El config es un JSON flexible que varía según el widget_type
