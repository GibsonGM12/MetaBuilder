## Historia de Usuario

**Como** administrador,

**Quiero** crear un nuevo dashboard con nombre y descripción,

**Para** tener un contenedor donde agregar widgets de visualización de datos.

## Descripción

El administrador puede crear dashboards vacíos que servirán como contenedores para widgets. Cada dashboard tiene un nombre único, una descripción opcional y una configuración de layout. Solo los administradores pueden crear dashboards.

## Criterios de Aceptación

```gherkin
Scenario: Admin crea dashboard exitosamente
  Given que soy un usuario con rol Admin
  When envío un POST a /api/dashboards con nombre "Ventas Q1" y descripción "Dashboard de ventas"
  Then se crea el dashboard con un UUID
  And la respuesta incluye id, name, description, created_at

Scenario: User no puede crear dashboard
  Given que soy un usuario con rol User
  When intento crear un dashboard
  Then recibo un error 403 Forbidden

Scenario: Nombre de dashboard es requerido
  Given que soy Admin
  When intento crear un dashboard sin nombre
  Then recibo un error 422 con mensaje descriptivo
```

## Notas Adicionales

- El dashboard se crea vacío (sin widgets)
- layout_config por defecto: `{"cols": 12, "rowHeight": 80}`
- Se almacena el `created_by` con el ID del usuario que lo crea
